22 Information
23 -----------
24 
25 CheMPS2 is a scientific library which contains a spin-adapted implementation
26 of the density matrix renormalization group (DMRG) for ab initio quantum
27 chemistry. This method allows to obtain numerical accuracy in active spaces
28 beyond the capabilities of full configuration interaction (FCI):
29 
30 * up to 40 electrons in 40 orbitals for general active spaces
31 * up to 100 electrons in 100 orbitals for one-dimensional active spaces, such as the pi-system of all-trans polyenes
32 
33 In addition, DMRG allows to obtain the 2-RDM of the active space efficiently.
34 The method is therefore ideal to replace the FCI solver in the complete active
35 space configuration interaction (CASCI) and complete active space self
36 consistent field (CASSCF) methods, when the active space sizes become
37 prohibitively expensive for FCI. The corresponding methods are called DMRG-CI
38 and DMRG-SCF, respectively. Because DMRG can handle the abovementioned active
39 space sizes, it allows to obtain FCI energies for small systems such as dimers,
40 while for larger systems it is ideal to treat the static/strong correlation in
41 a large active space.
42 
43 The design philosophy for CheMPS2 is to be a lightweight, efficient, and
44 stable library. For an input Hamiltonian and targeted symmetry sector, the
45 library performs successive DMRG sweeps according to a user-defined
46 convergence scheme. As output, the library returns the minimal encountered
47 energy as well as the 2-RDM of the active space. With the latter, various
48 molecular properties can be calculated, as well as the gradient and Hessian
49 for orbital rotations or nuclear displacements. In addition, several
50 correlation functions can be obtained to investigate the electronic structure
51 in the active space.
52 
53 To gain a better understanding of how to perform DMRG calculations for
54 large active spaces, you are encouraged to read the
55 [user manual](http://sebwouters.github.io/CheMPS2/index.html) and the
56 [three papers](#how-to-acknowledge-chemps2) listed below.
57 
58 Incorporation of the library into other codes is very simple due a
59 minimal API, as well as a python interface. Direct usage of the
60 library is illustrated in
61 
62 * the [c++](#4-test-libchemps2) tests
63 * the [python](#7-test-pychemps2) tests
64 * the [binary](#5-test-the-chemps2-binary) example
65 
66 The interfaces to [psi4](http://www.psicode.org)
67 and [pyscf](https://github.com/sunqm/pyscf) are described in the
68 [user manual](http://sebwouters.github.io/CheMPS2/index.html).
69 
70 CheMPS2 is parallelized for shared memory architectures with the Open
71 Multi-Processing ([OpenMP](http://openmp.org/wp/)) API and for
72 distributed memory architectures with the Message Passing Interface
73 ([MPI](http://www.mpi-forum.org/)). A hybrid combination of both
74 parallelization strategies is supported. In the future,
75 the calculation of the 2-RDM of the active space will be extended
76 to the 3-RDM.

134 Installation
135 ------------
136 
137 ### 1. Dependencies
138 
139 CheMPS2 can be built with [CMake](http://www.cmake.org/) and depends on
140 
141 - BLAS
142 - LAPACK
143 - GSL ([GNU Scientific library](http://www.gnu.org/software/gsl/))
144 - HDF5 ([Hierarchical Data Format Release 5](http://www.hdfgroup.org/HDF5/))
145 
146 It is parallelized for shared memory architectures with the Open
147 Multi-Processing ([OpenMP](http://openmp.org/wp/)) API and for
148 distributed memory architectures with the Message Passing Interface
149 ([MPI](http://www.mpi-forum.org/)). A hybrid combination of both
150 parallelization strategies is supported.
151 
152 ### 2. Download
153 
154 It is advised to clone the CheMPS2 git repository from github. In your
155 terminal, run:
156 
157  > cd /sourcefolder
158  > git clone 'https://github.com/sebwouters/chemps2'
159  > cd chemps2
160 
161 That way, future updates and bug fixes can be easily pulled in:
162 
163  > cd /sourcefolder/chemps2
164  > git pull
165 
166 ### 3. Build the chemps2 library and binary
167 
168 The files
169 
170  /sourcefolder/chemps2/CMakeLists.txt
171  /sourcefolder/chemps2/CheMPS2/CMakeLists.txt
172  /sourcefolder/chemps2/tests/CMakeLists.txt
173  /sourcefolder/chemps2/sphinx/CMakeLists.txt
174 
175 provide a minimal compilation. In your terminal, run:
176 
177  > cd /sourcefolder/chemps2
178  > mkdir build
179  > cd build
180 
181 CMake generates makefiles based on the user’s specifications:
182 
183  > CXX=option1 cmake .. -DMKL=option2 -DCMAKE_INSTALL_PREFIX=/option3 -DWITH_MPI=option4
184 
185 1. Option1 is the `c++` compiler; typically `g++`, `icpc`, or `clang++`
186  on Linux. It is advised to use the intel compiler if available.
187 2. Option2 can be `ON` or `OFF` and is used to switch on the intel math
188  kernel library.
189 3. /option3 is the prefix of the installation directory; typically `/usr`
190  or `/usr/local` on Linux. On my computer, libchemps2 is then installed
191  in `/option3/lib/x86_64-linux-gnu`, the headers in
192  `/option3/include/chemps2`, and the binary in
193  `/option3/bin/chemps2`.
194 4. Option4 can be `ON` or `OFF` and is used to switch on the possibility
195  to compile with MPI. Please note that the compiler should then provide
196  `mpi.h`. Option1 should hence be the `mpic++` compiler; typically
197  `mpic++` or `mpiicpc` on Linux. It is advised to use the intel compiler
198  if available.
199 
200 If one or more of the required libraries are not found, use the command
201 
202  > CMAKE_INCLUDE_PATH=option5 CMAKE_LIBRARY_PATH=option6 CXX=option1 cmake .. -DMKL=option2 -DCMAKE_INSTALL_PREFIX=/option3 -DWITH_MPI=option4
203 
204 instead, where option5 and option6 are respectively the missing
205 colon-separated include and library paths:
206 
207  CMAKE_INCLUDE_PATH=/my_libs/lib1/include:/my_libs/lib2/include
208  CMAKE_LIBRARY_PATH=/my_libs/lib1/lib:/my_libs/lib2/lib
209 
210 For debian/sid, the HDF5 headers are located in the folder
211 `/usr/include/hdf5/serial`. If CMake complains about the HDF5 headers,
212 try to pass it with the option
213 `-DHDF5_INCLUDE_DIRS=/usr/include/hdf5/serial`.
214 
215 To compile, run:
216 
217  > make
218 
219 To install, run:
220 
221  > make install
222 
223 For non-standard installation directories, please remember to append the
224 library path to `LD_LIBRARY_PATH` in your `.bashrc`.
225 
226 ### 4. Test libchemps2
227 
228 To test libchemps2 for compilation **without MPI**, run:
229 
230  > cd /sourcefolder/chemps2/build
231  > make test
232 
233 To test libchemps2 for compilation **with MPI**, run:
234 
235  > cd /sourcefolder/chemps2/build/tests
236  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./test1
237  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./test2
238  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./test3
239  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./test4
240  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./test5
241  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./test7
242  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./test10
243 
244 `YYY` specifies the number of threads per process and `ZZZ` the number of
245 processes. The tests only require a very limited amount of memory (order
246 100 MB). Note that the tests are too small to see (near) linear scaling
247 with the number of cores, although improvement should still be noticeable.
248 
249 ### 5. Test the chemps2 binary
250 
251 To test the chemps2 binary for compilation **without MPI**, run:
252 
253  > cd /sourcefolder/chemps2/build/CheMPS2
254  > ./chemps2 --help
255  > ./chemps2 --fcidump=/sourcefolder/chemps2/tests/matrixelements/H2O.631G.FCIDUMP \
256  --group=5 \
257  --multiplicity=1 \
258  --nelectrons=10 \
259  --irrep=0 \
260  --sweep_d=200,1000 \
261  --sweep_econv=1e-8,1e-8 \
262  --sweep_maxit=2,10 \
263  --sweep_noise=0.05,0.0 \
264  --twodmfile=2dm.out \
265  --print_corr \
266  --reorder=6,5,4,3,2,1,0,7,8,9,10,11,12
267 
268 To test the chemps2 binary for compilation **with MPI**, prepend the binary with:
269 
270  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./chemps2 [OPTIONS]

272 ### 6. Build PyCheMPS2
273 
274 PyCheMPS2 is a python interface to libchemps2, for
275 compilation **without MPI**. It can be built with
276 [Cython](http://cython.org/). The installation is independent of
277 CMake and assumes that you have installed the CheMPS2 library with
278 `make install`. For non-standard installation directories of CheMPS2,
279 please remember to append the library path to `LD_LIBRARY_PATH` in
280 your `.bashrc`. In addition, the include path should be appended
281 to `CPATH`:
282 
283  > export CPATH=${CPATH}:/option3/include
284 
285 where `/option3` is the option provided to CMake with
286 `-DCMAKE_INSTALL_PREFIX=/option3` above. For debian/sid, the HDF5
287 headers are located in the folder `/usr/include/hdf5/serial`. If it
288 was explicitly passed to CMake, it should also be appended to `CPATH`:
289 
290  > export CPATH=${CPATH}:/option3/include:/usr/include/hdf5/serial
291 
292 The python wrapper can be installed with:
293 
294  > cd /sourcefolder/chemps2/PyCheMPS2
295  > python setup.py build_ext -L ${LD_LIBRARY_PATH}
296  > python setup.py install --prefix=/option3
297 
298 On my machine, the python wrapper is installed to the folder
299 `/option3/lib/python2.7/site-packages`, but the folder `lib` and
300 the distribution of python can vary.
301 
302 Compilation of PyCheMPS2 occurs by linking to the `c++` library in the
303 installation directory. The installation of PyCheMPS2 will fail if that
304 library is not properly installed. If you have pulled a newer version of
305 CheMPS2, please remember to reinstall the `c++` library first, before
306 reinstalling PyCheMPS2!

309 ### 7. Test PyCheMPS2
310 
311 When libchemps2 has been compiled **without MPI**, PyCheMPS2 can be tested
312 by running (remember that the python site-packages folder can vary):
313 
314  > cd /sourcefolder/chemps2/PyCheMPS2/tests
315  > export PYTHONPATH=${PYTHONPATH}:/option3/lib/python2.7/site-packages
316  > python test1.py
317  > python test2.py
318  > python test3.py
319  > python test4.py
320  > python test5.py
321  > python test6.py
322  > python test7.py
323  > python test8.py
324  > python test9.py
325  > python test10.py
326 
327 If you compiled the `c++` library with `-DMKL=ON`, you might get the error
328 
329  Intel MKL FATAL ERROR: Cannot load libmkl_avx.so or libmkl_def.so.
330 
331 This issue of using Intel’s MKL inside python is known and reported. To
332 get the python tests to run, you can set the variable `LD_PRELOAD` in
333 order to preload libmkl\_rt.so. On my system, this is done with
334 
335  > export LD_PRELOAD=/opt/intel/mkl/lib/intel64/libmkl_rt.so
336 
337 The python tests do exactly the same thing as the `c++` tests above, and
338 illustrate the usage of the python interface to libchemps2. The tests
339 should end with a line stating whether or not they succeeded. They only
340 require a very limited amount of memory (order 100 MB). Note that the
341 tests are too small to see (near) linear scaling with the number of cores,
342 although improvement should still be noticeable.

767 (sigma bonding and antibonding), as well as one B2g, B3g, B2u, and B3u orbital
